# Cursor モデル選定方針

> **resource_type:** context  
> **resource_id:** context:model-policy  
> **owner:** PM のみ更新  
> **制定:** 2026-07-07  
> **改定:** 2026-07-07（閾値内=最適 · 超過=削減 · 日次監視）  
> **次回見直し:** 2026-08-07（月次）· 閾値改定時は随時

User 目標: **毎日 2〜3 時間** の URMS 開発を Cursor Pro 枠内で継続する。

## 運用モード（PM 判断 · 2 態）

| モード | 条件 | モデル方針 |
|--------|------|------------|
| **通常** | 下記「閾値」すべて **未満** | **用途に最適なモデルを PM が選定**（品質優先） |
| **削減 L1** | いずれか 1 つ該当 | Fast 中心 · Thinking 禁止 · スコープ半分 |
| **削減 L2** | いずれか 1 つ該当 | Fast + Auto のみ · Codex 禁止 · 実装最小 |
| **停止** | いずれか 1 つ該当 | セッション終了 · 翌日再開 or 軽作業のみ |

**原則:** 閾値を超える**まで**は最適モデルで進める。超えた**瞬間から**削減モードに切り替える（事前に常時 Fast にはしない）。

## 閾値（SSOT）

### 当日（日次監視）

| 閾値 | 削減モード |
|------|------------|
| 当日増分 **+8% 以上** | **削減 L1** |
| 当日増分 **+12% 以上** | **停止** |
| 1 セッションで **+5% 以上** 見込み（途中） | そのセッション以降 **削減 L1** |

### 月間（残枠）

| 閾値 | 削減モード |
|------|------------|
| 残枠 **40% 未満**（= 使用率 60% 超） | **削減 L1** |
| 残枠 **20% 未満**（= 使用率 80% 超） | **削減 L2** |
| 残枠 **10% 未満** | **停止** + Pro+ 検討を PM が提案 |

`usage-log.md` に **運用モード** を毎回記録する。

## 通常モード — タスク別 · 最適モデル（PM 選定）

User キーワードでは切り替えない。**PM がタスク種別から選ぶ。**

| 用途 | 最適モデル |
|------|------------|
| PM 日常（続けて · Go · status · Canvas） | Composer 2.5 Fast |
| 実装（domain / api / desktop） | **GPT-5.3 Codex** |
| 設計 · アーキ · 新境界 | **Claude Sonnet Thinking**（Plan → 承認 → Agent） |
| 探索 · 影響範囲 | Composer 2.5 Fast |
| デバッグ | Codex → **2 回失敗で** Sonnet Thinking |
| レビュー（diff 100 行超 · Go 前） | Sonnet Thinking（readonly） |
| セキュリティ / Bugbot | Sonnet Thinking（月 2 回まで · PM 起票） |
| 長文ドキュメント | Composer 2.5 Fast |

### 原則禁止（通常 · 削減 共通）

- **Opus / Thinking High** — ADR 級 + User 承認時のみ
- **Max Mode** — 禁止
- **1 ターンごとのモデル切替** — 禁止

## 削減モード — 実施内容

### 削減 L1

- モデル: **Composer Fast のみ**（Thinking · Codex 停止）
- スコープ: **半分**（1 テーマ → 半ステップ）
- Subagent / 広域 explore 禁止
- 進捗報告に `⚠ 削減 L1` を明記

### 削減 L2

- モデル: **Auto または Composer Fast**
- 実装禁止 — context 更新 · Go · ドキュメント 1 行のみ
- 進捗報告に `⚠ 削減 L2` を明記

### 停止

- その日の Agent 作業終了
- usage-log に理由を 1 行
- 翌日 **通常モード** 再開前に usage 再確認

### 通常モードへの復帰

- 当日増分が **+5% 未満** に落ち着いた翌日
- または新しい請求期間開始
- PM が `usage-log` に `通常` と記録してから最適モデルに戻す

## 日次監視（必須）

| タイミング | 内容 |
|------------|------|
| **セッション開始** | dashboard → 運用モード判定 → `usage-log.md` 追記 |
| **セッション途中** | +5% 見込みで L1 に切替 |
| **進捗報告** | 使用率 · 当日増分 · **運用モード** · 採用モデル |
| **セッション終了** | 再計測 · 翌日モードを記録 |

記録正本: [usage-log.md](./usage-log.md)

数値未確認時: **削減 L1 相当**（Fast のみ）で保守運転。推測値は記録しない。

## 進捗報告フォーマット（PM 必須）

正本: **[progress-report.md](./progress-report.md)** — 8 セクション · SSOT 同時更新 · **適宜見直し**

チャット報告末尾の Cursor 枠（要約）:

```
- 使用率 / 当日増分 / 運用モード / 本セッション模型 / 次セッション
```

詳細テンプレ · Canvas 同期 · 見直しトリガーは progress-report を参照。

## 1 日 2〜3 時間（通常モード時）

| 時間 | 最適模型 | 内容 |
|------|----------|------|
| 0:00–0:10 | Composer Fast | usage · モード判定 · 1 テーマ |
| 0:10–1:30 | **Codex** | 実装 + テスト |
| 1:30–1:45 | Composer Fast | dev:verify |
| 1:45–2:00 | Composer Fast | Go · usage-log |

設計変更日のみ Thinking を Plan に挿入（通常モード · 週 1〜2 回目安）。
